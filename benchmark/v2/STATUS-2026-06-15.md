## Known issues going into AWS day

1. **`BENCH_DURATION` env override does not apply to scenarios that hardcode `duration_seconds=`** in their `run()` (S-04, S-05, S-08, S-09, S-11). Not blocking for AWS — these scenarios have sensible hardcoded durations. To override, edit the scenario's constant or use `BENCH_VUS` only.
2. **`timeout_seconds: 60` in YAML may be tight for S-03 streaming under high concurrency.** S-03 on Mac+LiteLLM saw 11/11 stream_timeouts at 11 VUs. On AWS with a real Nexus + faster network, behavior may differ. If S-03 still saturates, raise to 120 in `config/*.yaml`.
3. **Connection-stage hook 403s manifested mid-session today** when our runner.py used a digit-only nonce — the pii-scanner phone regex (`\b\d{3}[-.\s]?\d{4}\b`) matched. Fixed via letter-heavy `secrets.token_urlsafe(8)`. Worth noting for the Nexus team: the regex is broad enough that any 7-digit number with one optional separator matches.
4. **Cache-hit telemetry only works for `X-Nexus-Cache` and `x-cache-status` header names.** If the new AMI emits a different header, update `runner.py` lines ~200, ~233.
5. **`--resume` not implemented.** Long runs (S-05 30-min soak) lose data if interrupted. Mitigation: don't interrupt; if needed, scenario-level resumption is left to AWS-day if it comes up.

python cli.py preflight

# Final report
python cli.py report --results-dir ./results/aws --format markdown
