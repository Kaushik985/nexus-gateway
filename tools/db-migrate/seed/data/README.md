# Seed baseline

`seed-baseline.sql` is a `pg_dump --data-only --column-inserts --disable-triggers`
snapshot of an operational source database with sensitive columns redacted to
a literal placeholder. `tools/db-migrate/seed/seed.ts` replays it under a
transaction, then:

1. Overwrites every Credential row's `encryptedKey` / `encryptionIv` /
   `encryptionTag` with a fresh AES-256-GCM encryption of a fake plaintext
   using the local `CREDENTIAL_ENCRYPTION_KEY` so rows decrypt locally.
2. Resets `Thing.status` to `offline` for any row left online/enrolled by the
   snapshot, so a fresh local boot doesn't misleadingly show stale services.

The file is small on purpose (~500 KB) — every high-cardinality event
table and every rollup is excluded at dump time, so what ships is purely
the static configuration shape (orgs, providers, credentials, routing
rules, hook configs, Things, alert rules) the app needs to boot.

## Excluded tables (snapshot keeps them empty)

| Reason | Tables |
|---|---|
| High-cardinality event tables (regenerated by traffic) | `traffic_event`, `traffic_event_payload`, `traffic_event_normalized` |
| Per-Thing time-series rollups (rebuilt by Hub jobs; E51 rename) | `thing_metric_rollup_5m`, `thing_metric_rollup_1h`, `thing_metric_rollup_1d`, `thing_metric_rollup_1mo` |
| Legacy ops-side rollups (kept for back-compat — drop after table removal) | `metric_rollup_5m/1h/1d/1mo`, `metric_ops_raw`, `metric_ops_rollup_1h/1d/1mo`, `rollup_watermark` |
| Scheduler / config history (large, env-specific) | `job_run`, `config_change_event` |
| Admin audit log | `AdminAuditLog` |
| Session tokens (not reusable across envs) | `RefreshToken`, `RevokedToken`, `ScimToken` |
| Diagnostic events (large, env-specific) | `thing_diag_event` |
| Prisma internal | `_prisma_migrations` (managed by `prisma migrate`) |
| One-off backfill scratchpads (not in schema.prisma) | `_e50_backfill_cursor` |

## Sensitive data

Real API-key ciphertexts are **never** committed. The dump retains Credential
row metadata (id, providerId, FK targets) but `encryptedKey`, `encryptionIv`,
`encryptionTag` are replaced with the literal placeholder
`'REDACTED_BY_PIPELINE'`. `seed.ts` overwrites these at the end of every
seed run with fresh local-key-encrypted ciphertext, so the rows decrypt
locally.

## Regenerating the snapshot (maintainer task)

The post-dump cleanup is currently inline (the standalone helper scripts
referenced in older revisions of this README do not exist in the repo).
Use the block below verbatim — it produces a byte-identical result to
the file checked in.

```bash
# 1. Dump from the operational source database. The exclusion list below
#    MUST match the table set in the table above — adding a new
#    high-cardinality table to the source without adding it here is how
#    the snapshot balloons unnoticed.
ssh ${SOURCE_DEPLOY_USER}@<source-host> "PGPASSWORD=<...> pg_dump -h localhost -U nexus -d nexus_gateway \
  --data-only --column-inserts --no-owner --no-privileges --disable-triggers \
  --exclude-table-data=public.traffic_event \
  --exclude-table-data=public.traffic_event_payload \
  --exclude-table-data=public.traffic_event_normalized \
  --exclude-table-data=public.thing_metric_rollup_5m \
  --exclude-table-data=public.thing_metric_rollup_1h \
  --exclude-table-data=public.thing_metric_rollup_1d \
  --exclude-table-data=public.thing_metric_rollup_1mo \
  --exclude-table-data=public.metric_rollup_5m \
  --exclude-table-data=public.metric_rollup_1h \
  --exclude-table-data=public.metric_rollup_1d \
  --exclude-table-data=public.metric_rollup_1mo \
  --exclude-table-data=public.metric_ops_raw \
  --exclude-table-data=public.metric_ops_rollup_1h \
  --exclude-table-data=public.metric_ops_rollup_1d \
  --exclude-table-data=public.metric_ops_rollup_1mo \
  --exclude-table-data=public.rollup_watermark \
  --exclude-table-data=public.job_run \
  --exclude-table-data=public.config_change_event \
  --exclude-table-data='public.\"AdminAuditLog\"' \
  --exclude-table-data='public.\"RefreshToken\"' \
  --exclude-table-data='public.\"RevokedToken\"' \
  --exclude-table-data='public.\"ScimToken\"' \
  --exclude-table-data=public.thing_diag_event \
  --exclude-table-data=public._prisma_migrations \
  --exclude-table-data=public._e50_backfill_cursor" > /tmp/dump.sql

# 2. Apply three post-dump cleanups (redact + strip psql meta + drop the
#    residual sequence-SET for the excluded _e50_backfill_cursor table —
#    --exclude-table-data only suppresses row INSERTs, not setval lines).
python3 - <<'PY'
import re, pathlib
src = pathlib.Path('/tmp/dump.sql').read_text()

# (a) Redact Credential ciphertext columns (positions 4, 5, 6).
pat = re.compile(
    r'(INSERT INTO public\.\"Credential\" \([^)]*\) VALUES \()'
    r"('[^']*', '[^']*', '[^']*', )"
    r"'[^']*', '[^']*', '[^']*'"
)
src, n = pat.subn(
    lambda m: m.group(1) + m.group(2) + "'REDACTED_BY_PIPELINE', 'REDACTED_BY_PIPELINE', 'REDACTED_BY_PIPELINE'",
    src
)
print(f"redacted {n} Credential row(s)")

# (b) Strip pg_dump 17 psql meta-commands (node-pg client.query rejects them).
src, n = re.subn(r'(?m)^\\(restrict|unrestrict)\s+.*\n', '', src)
print(f"stripped {n} psql meta-command(s)")

# (c) Drop any residual _e50_backfill_cursor sequence SET line + its comment.
lines = src.split('\n')
out, n = [], 0
i = 0
while i < len(lines):
    if '_e50_backfill_cursor' in lines[i]:
        while out and (out[-1].startswith('--') or out[-1] == ''):
            out.pop()
        n += 1; i += 1
        while i < len(lines) and (lines[i] == '' or lines[i].startswith('SELECT') or
                                   (lines[i].startswith('--') and 'Name:' not in lines[i] and 'PostgreSQL' not in lines[i])):
            if '_e50_backfill_cursor' in lines[i]: n += 1
            i += 1
        continue
    out.append(lines[i]); i += 1
pathlib.Path('/tmp/dump.sql').write_text('\n'.join(out))
print(f"stripped {n} e50 residual line(s)")
PY

# 3. Sanity-check: no real ciphertext leaked, ~15 redaction markers (5 rows × 3 cols).
grep -c "REDACTED_BY_PIPELINE" /tmp/dump.sql       # expect 15
grep -c "_e50_backfill_cursor" /tmp/dump.sql       # expect 0

# 4. Install.
mv /tmp/dump.sql tools/db-migrate/seed/data/seed-baseline.sql
```
